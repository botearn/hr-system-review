# 需求收集 · HR System Cloud V2

> 内部团队用。**提需求 ≤ 2 分钟**；细节由产品在评审后补全。  
> V2 重点：面试题、面试反馈、面试官采集信息。

---

## 怎么提

1. 打开 `功能需求收集模板.xlsx` → **需求池**，从第 4 行起填（第 3 行是示例）
2. 必填 5 项：**标题 · 问题 · 期望 · 验收 · 优先级**
3. 状态保持 `待评审`，周会统一过

**一条好需求 = 说清楚痛点 + 做完长什么样 + 怎么验**

---

## 填写模板（复制到表格）

```
标题：【动词 + 对象 + 结果】
      例：面试官在线提交反馈并自动写入候选人 Timeline

问题：【谁】在【什么场景】下【现在多痛苦】
      例：顾问每月抄录 30 场飞书反馈，单场约 15 分钟，格式不统一

期望：【做完后系统/流程变成什么样】
      例：面试官系统内填结构化表，提交后 Timeline 自动出现节点

验收：【2–3 条，Given / When / Then】
      1. Given 已安排一面  When 面试官提交  Then Timeline 有反馈节点
      2. Given 非参与面试官  When 打开候选人  Then 不可见完整简历

优先级：P0 / P1 / P2 / P3
模块：  面试题 | 面试反馈 | 面试流程 | 候选人 | 岗位 | 匹配 | 看板 | 其他
规模：  S(≤3天) / M(3–10天) / L(>10天，需拆分)
```

---

## 优先级（团队共识）

| 级别 | 什么时候用 |
|------|------------|
| **P0** | 阻断业务 / 数据错误 / 权限问题 → 随时插队 |
| **P1** | V2 必做，不做上不了线 |
| **P2** | 明显提效，可手工顶一阵 |
| **P3** | 有了更好，不承诺 |

---

## 模块速查（V2）

| 模块 | 典型需求 |
|------|----------|
| **面试题** | 题库、按岗位/轮次组卷 |
| **面试反馈** | 评分表、多面试官汇总、进 Timeline |
| **面试流程** | 排期、结果、状态自动推进、逾期提醒 |
| **权限** | interviewer 角色、可见范围 |
| **其他** | 匹配、看板、导出、飞书同步等 |

预设 Backlog 见 Excel「V2种子需求」Sheet，可直接改优先级后当正式需求。

---

## 状态流转

```
待评审 → 已采纳 → 开发中 → 待验收 → 完成
              ↘ 不做 / 暂缓
```

| 状态 | 含义 |
|------|------|
| 待评审 | 刚提交 |
| 已采纳 | 进版本计划，产品可补 FR/数据设计 |
| 开发中 / 待验收 / 完成 | 常规流转 |
| 不做 | 需写一句原因 |
| 暂缓 | 依赖未就绪 |

---

## 三条原则

1. **先问题，后方案** — 写「现在多痛」，少写「要用 React 做 xxx」
2. **能拆就拆** — 超过 M 规模或跨 2 个模块，开多条
3. **验收要可测** — 「更好用」不算，「提交后 3 秒内在 Timeline 可见」算
